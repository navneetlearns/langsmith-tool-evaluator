Eval Testing Principles (HEART.md)

  1. Think like a user — send the query, capture what comes back. Errors are real results, not bugs in the harness.
  2. Wait patiently — SSE timeout is 300s per query. Multi-turn, data-driven responses take time.
  3. Measure response time — every query records response_time_seconds. Shows which features users wait longest for.
  4. Versioned reruns — each run is query_results_v{N}.jsonl. Never overwrites. Compare across versions to track improvement.
  5. No retry, no masking — if a query fails, that IS the result. The eval's job is to surface truth.
  6. Full trace capture — tool calls, SSE status sequence, response text, suggestions, errors, timing — all recorded per query.
